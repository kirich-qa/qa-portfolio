## 📂 Проекты

### 🌌 Проект 1: Stellarium Mobile (Android)
Сравнительное тестирование мобильного приложения Stellarium Mobile версий 1.15.6 и 1.29.3.

В ходе тестирования проводился анализ:
- Пользовательского интерфейса
- Стабильности работы приложения
- Отображения карты неба и астрономических объектов
- Работы поиска объектов
- Масштабирования и навигации
- Отображения платного контента
- Работы спутников
- Отображения линии горизонта

**Тестовая документация:**
- [📄 Чек-лист](Stellarium_Checklist.pdf)
- [📄 Тест-кейсы (18 сценариев)](Stellarium_Test_Cases.pdf)
- [📄 Test Summary Report](Stellarium_Test_Summary_Report.pdf)
- [🐞 Баг-репорты (6 дефектов)](Stellarium_Bug_Reports.pdf)

**🐞Выявленные проблемы:**
- Смещение небесных объектов при увеличении масштаба
- Полоса интерфейса при повороте экрана
- Некорректное отображение линии горизонта
- Наложение названий небесных объектов 
- Периодические краши при повторном запуске
- Некорректное отображение названий h Персея и χ Персея

**📊 Результат:**
В ходе тестирования был проведён сравнительный анализ двух версий приложения Stellarium Mobile. Были выявлены различия в пользовательском интерфейсе, стабильности работы, отображении объектов и доступности функционала. Также были обнаружены дефекты, влияющие на отображение карты неба, работу интерфейса и стабильность запуска приложения, включая периодические краши при повторном запуске.

### ⚙️ Проект 3: Тестирование API (Postman)
Примеры тестирования REST API с использованием **Postman** на основе API Swagger Petstore.

**Покрытие тестирования:**
- **CRUD операции:** цепочка запросов POST, GET, PUT, DELETE
- **Проверка ответов:** статус-коды, время ответа и структура данных
- **Валидация данных:** проверка типов (string, number) и обязательных полей
- **Автоматизация тестов:** использование переменных окружения (Environment Variables) для базового URL и динамических данных между запросами
- **Негативный сценарий:** проверка обработки ошибки 404

**Файлы проекта:**
