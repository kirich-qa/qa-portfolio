## 📂 Проекты

### 📚 Проект 1: Интернет-магазин «Буквоед» 
Тестирование ключевого функционала e-commerce платформы: поиск товаров, фильтрация каталога, работа корзины и оформление заказа.

В ходе тестирования проверены:
- Главная страница и основные разделы сайта
- Поиск книг по названию и автору
- Поиск по неполному запросу
- Фильтрация по жанру и диапазону цен
- Сортировка товаров
- Карточка товара
- Работа корзины
- Расчёт итоговой суммы заказа
- Оформление заказа
- HTTP-запросы при добавлении товаров в корзину

**🐞Выявленные проблемы:**
- Некорректная работа фильтрации и поиска (нерелевантные результаты)
- Ошибки сортировки товаров по цене
- Проблемы в работе корзины (дублирование товаров, несоответствие ответа сервера фактическому состоянию)
- Отсутствие валидации обязательных полей при оформлении заказа
- Недочеты отображения данных (форматирование итоговой суммы)

**📊Результат:**

Проведено комплексное функциональное тестирование пользовательских сценариев, выявлены дефекты, влияющие на пользовательский опыт и корректность работы системы.

### 🌌 Проект 2: Stellarium Mobile (Android)
Сравнительное тестирование мобильного приложения Stellarium Mobile версий 1.15.6 и 1.29.3.

В ходе тестирования проводился анализ:
- Пользовательского интерфейса
- Стабильности работы приложения
- Отображения карты неба и астрономических объектов
- Работы поиска объектов
- Масштабирования и навигации
- Отображения платного контента
- Работы спутников
- Отображения линии горизонта

- Смещение небесных объектов при увеличении масштаба

Были выявлены различия в пользовательском интерфейсе, стабильности работы, отображении объектов и доступности функционала. Также были обнаружены дефекты, влияющие на отображение карты неба, работу интерфейса и стабильность запуска приложения, включая периодические краши при повторном запуске.

### ⚙️ Проект 3: Тестирование API (Postman)
Примеры тестирования REST API с использованием **Postman** на основе API Swagger Petstore.

**Покрытие тестирования:**
- **CRUD операции:** цепочка запросов POST, GET, PUT, DELETE
- **Проверка ответов:** статус-коды, время ответа и структура данных
- **Валидация данных:** проверка типов (string, number) и обязательных полей
- **Автоматизация тестов:** использование переменных окружения (Environment Variables) для базового URL и динамических данных между запросами
- **Негативный сценарий:** проверка обработки ошибки 404
